# Maintainer Expectations

Maintainers are responsible for process quality, publication discipline, and repository health.

## Maintainer Responsibilities

- keep issue and PR triage current
- enforce the issue-first workflow
- require research before direction selection
- hold human approval gates for normative, semantic, and public-facing changes
- protect the boundary between public materials and unpublished working materials
- keep contribution and repository policy documentation current
- ensure build and validation paths remain functional

## Review Posture

Maintain standards quality by defaulting to:

- explicit rationale over implied intent
- compatibility analysis over intuition
- narrow, reviewable changes over bundled refactors
- decision logging over ephemeral chat history

## Maintenance Cadence

Recommended recurring maintenance:

- weekly or as-needed issue triage
- periodic review of stale proposals and unresolved approvals
- per-change verification that docs and templates still match actual workflow
- pre-release review of versioning and publication state

## Approval Duties

Maintainers decide:

- whether research is sufficient
- which direction is selected
- whether proposal text is ready for public review
- whether a fix is safe to integrate
- whether a release or draft submission should proceed

## Process Changes

Maintainers must explicitly review changes that affect:

- approval gates
- publication permissions
- decision log requirements
- role boundaries
- record retention policy

Changes that affect public-facing policy must be documented in this repository.
