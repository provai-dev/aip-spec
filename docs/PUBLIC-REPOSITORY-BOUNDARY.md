# Public Repository Boundary

This repository is public-facing. Unpublished working artifacts must not be committed here.

## Keep Out Of This Repository

- unpublished drafting notes
- internal review materials
- incomplete decision records
- issue-specific working notes that are not ready for publication
- approval bookkeeping that is not part of the public record

## Keep Public

- accepted proposal text
- merged spec edits
- public issue and PR discussion
- public rationale needed to understand the final result
- contributor and maintainer policy
- the `docs/aip-spec.md` compatibility page

## Repository Rule

Only final, human-approved outputs belong in this repository.

No local drafting practice reduces the amount of human review required for public changes.
